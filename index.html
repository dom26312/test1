TC Telnet Config Saver
Алгоритм Отчет Безопасность Запуск

Python-автоматизация для сетевого оборудования

Сохраняет конфигурацию только после проверки Shelf ID

Приложение проходит по IP-адресам из Excel, проверяет доступность, авторизуется по Telnet, сверяет Shelf ID и выполняет save conf только при полном совпадении.

Логика обработки

Понятный маршрут для каждого IP

Ошибка на одной плате не останавливает обработку остальных устройств. Каждый результат попадает в отчет и лог.

01

Ping

Если устройство недоступно, статус сразу фиксируется как «Недоступен (Ping)».

02

Telnet

Подключение к порту 23 и до трех попыток авторизации.

03

Shelf ID

Команда show shelf-id возвращает значение для сравнения с Excel.

04

Save

save conf выполняется только при совпадении Shelf ID.

report.xlsx

Отчет показывает, что произошло с каждой платой

В итоговой таблице видны ожидаемый и полученный Shelf ID, финальный статус и короткий комментарий.

192.168.1.12 C-018 — Ping Устройство недоступно

Важные ограничения

Секреты не должны попадать в публичный репозиторий

Логин, пароль и адреса оборудования лучше хранить локально в конфиге или переменных окружения. На сайте опубликована только логика работы, без реальных учетных данных.

Защита от неверной платы

Несовпадение Shelf ID блокирует сохранение конфигурации и отправляет N на выходе.

Полная трассировка

Файл save_conf.log фиксирует ping, авторизацию, Shelf ID, logout и ошибки.

Продолжение при ошибках

Исключение на одном IP записывается в отчет, после чего обработка идет дальше.

Быстрый старт

Минимальная структура запуска

Подготовьте файл ip адреса.xlsx со столбцами IP Address и Shelf ID, установите зависимости и запустите скрипт. 

pip install pandas openpyxl telnetlib3
python save_conf.py
