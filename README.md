# Web Telnet Control site

Статический веб-интерфейс для GitHub Pages по PRD автоматического сохранения конфигурации плат по Telnet.

## Файлы

- `index.html` — веб-интерфейс загрузки Excel, запуска обработки, таблицы и лога.
- `styles.css` — адаптивный дизайн.
- `script.js` — чтение Excel на клиенте, demo-run, экспорт `report.xlsx` и `save_conf.log`.
- `enable-github-pages.ps1` — помощник для включения GitHub Pages через GitHub API.

## GitHub Pages

В репозитории откройте `Settings` → `Pages` и выберите:

- Source: `GitHub Actions`

Workflow `.github/workflows/pages.yml` опубликует статический сайт из корня репозитория.
После публикации сайт будет доступен по адресу GitHub Pages для репозитория.

## Безопасность

Не публикуйте реальные логины, пароли и IP-адреса оборудования в публичном репозитории. Храните их локально в конфиге или переменных окружения.

## Ограничение GitHub Pages

Браузерная страница на GitHub Pages не может выполнять настоящий Ping и Telnet к оборудованию.
Для боевого режима нужен локальный backend/агент, который будет принимать загруженный Excel, выполнять сетевые команды и возвращать отчет.
